# Concise — Design System

**Concise** is an LLM-powered note-taking application — a cross-platform desktop
and tablet app that extracts content in a niri-like window/scrolling manager and
synthesises it into **Object-Concept-Mappings (OCM)** stored in an embedded
database. Those mappings build up a knowledge graph; notes get generated by
querying that graph.

The product is **file-first**, **largely open-source-model-powered**, and ships
as small native binaries (Tauri) on **Windows, macOS, Linux, Android, and iOS
tablets**. It includes a knowledge-graph + flashcard surface for recall.

The interface is deliberately quiet — near-black "carbon" surfaces, dense
translucent panels, almost no chrome — so the user's own content is what shows
up loudest on screen.

### Vocabulary lifted from the repo

The Figma MVP and the repo use overlapping but slightly different terms — keep
both in mind:

- **Fragment** — a unit of raw, uploaded source material (a lecture PDF, a
  board photo, a video). Lives in the **Fragment Store**.
- **Concept** — a structured, AI-distilled understanding extracted from one or
  more fragments. Has sub-items (definitions, theorems, proofs). Lives in the
  **Concept Store**.
- **Object-Concept-Mapping (OCM)** — the repo's term for the underlying
  representation: how concepts relate to their source objects. The knowledge
  graph is the set of these mappings.
- **Notes** — the user's own writing, organised hierarchically.

## Content Fundamentals

How copy is written in Concise.

**Voice — utilitarian, second-person, no hedging.**
The product talks _to the user_, not _about itself_. Section titles are nouns
("Concept Store", "Fragment Store", "Notes"). Prompts pose a single question
("What do you want to learn today?"). Dialog titles are imperative verbs
("Build Concept", "Add Fragments").

**Casing — Title Case for nouns + actions, Sentence case for help.**
- Title Case: `Concept Store`, `Build Concepts`, `Select Fragments`, `Add Fragment`
- Sentence case: `Select the fragments your concept will be built from.`
- Search placeholder is always exactly `Search...` (lowercase ellipsis, three dots, no Unicode).
- Field labels are short and Title Cased: `Name`, `Fragment`, `Description`.

**Punctuation — minimal.** No exclamation marks. Trailing periods on full-sentence
help text only; never on labels, buttons, or chip values. Ellipsis to denote a
truncated title (`Sequence and Conve…` in the store panel) is rendered with
CSS overflow, not typed.

**Pronouns.** "You" addresses the student directly. Never "we"; the app does not
speak for the team. Never refer to AI by name — actions read as if the student
performed them ("Build Concept", not "Generate with AI").

**Specific examples (verbatim from the file):**
- Title: **What do you want to learn today?**
- Long prompt seed: _"I want to learn more about limits, including the
  different types of limits in the respective metric, norm, and inner product
  space. Break it down into the following sections:"_
- Help under a dialog title: _"Select the fragments your concept will be built from."_
- Help under a dialog title: _"Upload a folder with files to process into fragments."_
- Field help: _"Describe what this fragment is."_, _"Describe the concept you want to extract from the fragment."_

**Domain register.** This is an academic tool. Math vocabulary appears uncut —
"metric spaces", "Cauchy sequence", "holomorphic" — and we don't apologise for it
or wrap it in tooltips. The audience knows what these words mean.

**No emoji.** None appear in the figma; none should appear in product copy.

**No hashtags, no marketing.** No exclamatory growth-y copy ("Get started in
seconds!"). The app is a workbench, not a landing page.

## Visual Foundations

The system is **97% greyscale on a single charcoal background**. Depth comes
from stacked white-with-low-alpha overlays, not from shadows or borders.

### Color

| Token | Value | Role |
|---|---|---|
| `--carbon-1` | `#0E0E0E` `rgb(14,14,14)` | App background — the only canvas color |
| `--white-046` | `rgba(255,255,255,0.046)` | Sidebar + panel fill |
| `--white-05`  | `rgba(255,255,255,0.05)`  | Inputs, hairline borders |
| `--white-096` | `rgba(255,255,255,0.096)` | Composer fill (the only "lifted" surface) |
| `--white-10`  | `rgba(255,255,255,0.10)`  | Button-top inner highlight, checkbox fill |
| `--white-15`  | `rgba(255,255,255,0.15)`  | Checkbox border |
| `--carbon-4`  | `#404040`                 | Solid CTA fill ("Build Concepts", "Add Fragment") |
| `--fg-0`      | `#FFFFFF`                 | Primary text + active icon |
| `--fg-3`      | `rgba(255,255,255,0.50)`  | Placeholders, list metadata |
| `--icon-mute` | `#A8A8A8`                 | Leading icons in inputs/panels |
| `--icon-base` | `#B3B3B3`                 | Button-internal glyphs |
| `--accent`    | `#E8B84A`                 | **Avatar ring only** — used once |

The accent is intentionally locked down. It is a single warm note in an
otherwise neutral system — never use it for buttons, links, or status. Status
colours (success/warning/danger/info) are defined for forms but unused in the
MVP figma.

### Type

**PP Mori** (Pangram Pangram) is the primary face. A contemporary grotesque
with tight spacing and a polished, slightly editorial flavour — closest free-
adjacent equivalent to Söhne. Licensed via Pangram Pangram's indie-friendly
model: free for personal/educational use, $50 for commercial single-weight
licenses (Concise qualifies for the indie tier).

Weights in active use:
- **Regular 400** — list subtitles, body copy, large prompts
- **Medium / Semibold 500–600** — panel items, button labels, dialog titles
- **Extrabold 700+** — panel section titles (`Concept Store`, etc.)

Until the trial files land in `fonts/`, the system **falls back to Inter**
gracefully — every CSS variable references `"PP Mori", "Inter"` in order, so
existing previews keep rendering with no visible breakage.

**Ranade** (Indian Type Foundry) is the **wordmark face only**. Heavy weight,
inktrap detailing, wrapped in dimmed curly braces. Never use Ranade in UI
copy — it's reserved for the brand mark.

**JetBrains Mono 400/500** carries inline math and code — disambiguated `0`
and `l`, x-height matching the body face.

Sizes lean small and dense:
- `12px / 16px` — primary UI text, list items
- `14px / 20px` — composer body (the only place the user types)
- `20px / 1.2` — dialog titles
- `24px / 1.2` — the "What do you want to learn today?" prompt

### Installing PP Mori

1. Visit [pangrampangram.com/products/mori](https://pangrampangram.com/products/mori)
2. Click "Download Trial" — provide an email; trial files arrive immediately.
3. Drop the following files into `fonts/`:
   - `PPMori-Regular.woff2` (or `.ttf`)
   - `PPMori-SemiBold.woff2`
   - `PPMori-Extrabold.woff2`
4. The `@font-face` blocks in `colors_and_type.css` pick them up automatically.

### Spacing & Layout

A 4px grid. Step values: 4, 6, 8, 10, 12, 16, 20, 24, 32, 40. The composer/panel
gap is 12px; the panel internal padding is also 12px; the app body padding is 16px.
There are no wide expanses — the design is intimate, like a desktop app sidebar.

Fixed widths: sidebar 60px, panel 180px, composer 514px. The right two-thirds
of the canvas is intentionally empty until the student starts a thread.

### Borders, Radii, Shadows

- **Radii:** 4 (checkbox), 6 (buttons/chips), 8 (search/composer), 12 (panels).
  Never larger; cards are crisp, not pillowy.
- **Borders:** 1px `--white-05` on the sidebar and the multi-select popover.
  Nothing else has a stroke. Edges come from background contrast.
- **Outer shadow:** essentially none. The app reads as flat planes of light.
- **Inner highlight:** the one elevation trick. Pressable controls (the solid
  "Build Concepts" / "Add Fragment" buttons) carry
  `inset 0 2px 4px rgba(255,255,255,0.10)` along the top edge — a faint
  "polished pebble" sheen that says _this is a thing you can press_.
- **Dialogs** sit on a scrim of `rgba(14,14,14,0.72)` and use a soft outer
  drop shadow only when truly floating.

### Backgrounds & Imagery

There are no decorative backgrounds, gradients, or illustrations. The only
imagery in the system is the **user's own avatar** (a circular crop of a real
photo) and **document thumbnails** the user uploads. Imagery is therefore warm
and human against the cool monochrome chrome — the contrast is the point.

No grain, no noise, no patterns. If a future surface needs warmth, reach for a
single user-supplied photo, not synthetic gradients.

### Transparency & Blur

Transparency is the system. Every "surface" is white-on-carbon at low alpha.
**Blur is not used** in the MVP — surfaces are layered as flat tints. If we
introduce overlay panels in the future, a `backdrop-filter: blur(20px) saturate(140%)`
would extend the system cleanly, but it is not present today.

### Motion

The system is quiet. Defaults:
- **Hover:** background steps up one alpha tier (e.g. `--white-046` → `--white-096`),
  no scale, no translate. Duration `90ms` `ease-out`.
- **Press:** the inner highlight flips to `inset 0 1px 2px rgba(0,0,0,0.20)` —
  the button "pushes in". No scale.
- **Popovers:** fade + 4px translateY-in. Duration `160ms` `cubic-bezier(0.16, 1, 0.3, 1)`.
- **Dialogs:** fade + 8px translateY-in. Duration `240ms`. The scrim fades over `200ms`.
- **No bounces.** Nothing overshoots.

### Hover / Press / Focus

- **Hover** on translucent surfaces: bump alpha by ~0.05.
- **Hover** on solid buttons: lift from `#404040` → `#525252`.
- **Press**: swap the inner top-highlight for an inner top-shadow (see above).
- **Focus**: thin 2px white ring at 85% alpha, no glow. Form fields show focus
  via border colour change to `--white-15`.
- **Disabled**: 35% alpha, no pointer events.

### Cards

A "card" in Concise is really a tinted panel:
- Background: `--white-046`
- Border: `1px solid --white-05` (sidebar) or none (panels)
- Radius: `12px`
- No shadow; the carbon background does the work.

There is no concept of an elevated/floating card with a heavy drop shadow. If
something needs to feel above the page (a dialog, a popover), it sits on a
scrim and earns its outer shadow.

### Layout rules

- The sidebar is fixed (60px) and the panel is fixed (180px). Width breakpoints
  are not part of the MVP — the file is desktop-only at 1280×832.
- The composer is centred horizontally in whatever space the panel leaves.
- Content is left-aligned inside panels; centred inside dialogs and on the
  empty composer canvas.

## Iconography

**Heroicons** is the entire icon system — `mini` (12/16px) inside inputs and
small lists, `outline` for the sidebar tabs, `solid` for the cog, and `micro`
(16px filled) for file-type badges in the Fragment Store. Every glyph in the
figma traces back to a Heroicons component name (e.g.
`heroicons-mini/magnifying-glass`, `heroicons-solid/cog-6-tooth`).

We've copied the actual SVGs the file uses into `assets/icons/` so they can be
served locally. For anything not in the file but needed downstream
(e.g. `ellipsis-horizontal`, `x-mark`, `sparkles`, `chevron-right`), we lifted
the canonical paths from the public Heroicons set, same family, same weights —
flagged here so you know they were not literally in the figma.

**Sizing conventions** (lifted from the figma):
- Inside `28px` inputs → 12px icons
- Inside `36px` sidebar tabs → 24px outline icons
- Inside `24px` buttons → 12px mini icons
- File-type badges in the Fragment Store → 16px mini/micro filled

Stroke icons run at Heroicons' default 1.5px stroke. Filled icons inherit
`currentColor`. **No icon ever carries colour** beyond the foreground scale —
muted icons use `--icon-mute` or `--icon-base`; active icons use `--fg-0`.

**No emoji.** **No Unicode character icons** (no `→`, no `★`). Use SVG.

**Avatar.** Real photograph, circular crop, 36px in the sidebar. No fallback
initials block was specified — if the avatar is missing, render a `--white-10`
disc with no glyph.

## Caveats — please confirm

1. **Logo image not yet imported.** The repo has `public/concise.png` (a galaxy
   spiral mark, ~1.8MB) but the GitHub access rotated mid-session before I could
   fetch it. The wordmark page currently shows the `{Concise.}` lockup as the
   primary mark. If the spiral is meant to live alongside the wordmark, drop
   the PNG into `assets/` and I'll wire it in. Otherwise, the lockup is enough.
2. **Mono pairing.** Concise is a math-heavy product so a monospaced face
   matters. I picked **JetBrains Mono** for its disambiguated glyphs and the
   DM-Sans-matching x-height. Open to **Geist Mono** or **IBM Plex Mono**.
3. **Accent colour.** The only non-grey in the Figma is the warm-gold avatar
   ring. I lifted that as `--accent` (#E8B84A) and reserved it for emphasis —
   the wordmark dot and the avatar ring use it, nothing else does. If Concise
   has an actual brand colour (the galaxy spiral might suggest something),
   swap it in `colors_and_type.css`.
4. **Icon font files.** All SVGs are served locally from `assets/icons/`. A
   few icons (`ellipsis-horizontal`, `x-mark`, `sparkles`, `chevron-right`)
   were not in the Figma — canonical Heroicons paths filled the gap.
5. **Base UI vs Headless UI.** The repo uses Base UI (the new package from the
   Headless UI team). Same API shape, same interaction quality — the UI kit's
   `MultiSelect`/`Dialog` components mimic the visual contract, not the actual
   primitives. Swap in real `@base-ui-components/react` when integrating.
6. **Knowledge-graph + flashcard surfaces** are mentioned in the repo README
   but not in the Figma yet. They'll need their own visual decisions — graph
   nodes/edges treatment, card flip motion, recall states. Flag when ready
   and I'll extend the kit.
7. **niri-like window scrolling manager** — the repo describes the content
   extraction surface as a horizontally-scrolling window manager (after the
   [niri](https://github.com/YaLTeR/niri) Wayland compositor). That UX isn't
   in the Figma; treat it as another future surface.
8. **Status colours unused in figma.** I defined `--success/--warning/--danger/
   --info` for completeness but they are not validated against actual Concise
   states. Confirm before shipping.

## How to use this system

1. Include `colors_and_type.css` once, add `class="concise"` to your `<body>`.
2. Reach for the CSS variables, not the raw hex values — the whole point is
   that adjusting `--white-046` once re-skins every panel.
3. For React surfaces, copy components from `ui_kits/web_app/` and edit. They
   are deliberately cosmetic — there is no production logic. Treat them as a
   visual contract.
4. When in doubt, the answer is "make it smaller, quieter, and remove the border".
